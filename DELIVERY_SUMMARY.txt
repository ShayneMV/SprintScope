## How to Run

### Step 1: Configure Paths
Edit `_app/config.toml`:
```toml
[paths]
data_root = "C:\Users\YOUR_NAME\OneDrive - Team Saudi\Documents\Athletics\Laveg\Laveg script"
```

### Step 2: Install Dependencies
```bash
cd /workspace/laveg_app
pip install -r requirements.txt
```

### Step 3: Start the App
```bash
streamlit run _app/app.py
```

Opens at `http://localhost:8501`

### Step 4: Import Trials
1. Place SprintScope CSV files in event group folders (sprints/, hurdles/, etc.)
2. Click "🔄 Scan for new CSV files" in the sidebar
3. App automatically computes metrics and imports to database

## The Four Tabs

1. **Overlay Plot** — Compare velocity profiles (velocity vs. filtered distance)
   - Multiple trials overlaid on same chart
   - Peak velocity marked with stars
   - Short trials shown as dashed lines
   - Download as PNG

2. **Split Comparison** — See splits at any interval (5m, 10m, 20m, 50m)
   - Rows: split distances
   - Columns: split time, cumulative time, velocity for each trial
   - Fastest cumulative time highlighted with 🔥
   - Download as CSV

3. **Development Trend** — Track athlete progress over time
   - Metric selector (peak V, time to 10m, time to 30m, velocity at distance, etc.)
   - X-axis: date, Y-axis: metric value
   - Line chart with markers, summary stats
   - Download as CSV or PNG

4. **Trial Detail** — Deep dive into a single trial
   - Full metadata block (athlete, date, surface, wind, etc.)
   - Device split table (what the device recorded)
   - Velocity curve (distance vs. velocity)
   - Custom splits table (computed at selected interval)
   - Download any table as CSV

## Database Schema

### `trials` table
- Core metadata: athlete, test ID, date, distance, surface, etc.
- Computed metrics: peak_v_ms, peak_v_distance_m, split_origin_t_s
- Flags: flag_short_trial (1 if undershoot, 0 otherwise)
- Audit trail: distance_meta_m (original parsed), distance_source (metadata/estimated/user/missing)

### `samples` table
- Time series at 200 Hz: time, distance (raw + filtered), velocity, acceleration
- 3000+ rows per trial, indexed by (trial_id, t_s)

### `device_splits` table
- Pre-computed splits from device: split distance, delta time, cumulative time, velocity, is_peak_marker flag
- 6-8 rows per trial

**Database Location**: `%LOCALAPPDATA%\laveg_app\laveg.sqlite` (OUTSIDE OneDrive to prevent corruption)

## Architecture Highlights

1. **Pure Functions**: All metrics in splits.py are pure (no side effects, fully testable)

2. **Type Hints**: Throughout for clarity and IDE support

3. **Error Handling**: Graceful quarantine of malformed files, no crashes

4. **Deduplication**: File hash + (test_id, athlete_raw) prevents duplicate imports

5. **Configuration**: Externalized in config.toml, sensible defaults auto-applied

6. **Separation of Concerns**:
   - Parser handles format
   - Splits handles computation
   - DB handles persistence
   - App handles UI

## Troubleshooting

| Issue | Solution |
|-------|----------|
| "No event groups found" | Check data_root has subfolders (sprints/, hurdles/, etc.) |
| "Parser rejected file" | Ensure first line is `# SprintScope Test Export` |
| "Database locked" | Close other app instances; check db_path not in OneDrive |
| "No trials imported" | Place CSVs in event group subfolders, not data_root directly |
| Module not found error | Make sure running from `/workspace/laveg_app` directory |
